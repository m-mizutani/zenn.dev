# 小規模セキュリティチームにおける監視の辛さ：大量の放置アラート

- セキュリティアラートが蓄積してしまう
  - 各方面からアラートが飛んでくる
    - それ自体はよいこと
  - 内容を確認＆トリアージがどうしても後手後手になる
    - アラートの内容確認するのも手間がかかる
    - 優先するタスクがたくさんあって劣後する
      - 仕事的にコンテキストスイッチするだけでも大変
    - 結果、アラートがたまる
      - たまるとさらに見たくなくなってくる
      - 結果、数百、数千のアラートが対応されずに放置される
- せっかくセキュリティ監視をしているのにまったく無意味になってしまう
  - 専任チームがいれば順次アラート処理してくれるのに…
    - 専任のアナリストがいればコンテキストスイッチ無しで対応できる
    - しかしほとんどの組織ではそんな人員はいない
    - Ubieも同様で、チームとしては3人いるが全員兼任になっている
  - 1000件の中に1件真にヤバいやつがあるかもしれない
    - 昨今のランサムウェア被害などをみるとひやひやしませんか？

# 本質的な苦しみは何か？

- では本質的なペインはなんなのか？
- そこを取り除くとアラートが消化されるのではないか

## 調査のための方法を思い出すのが大変

- まず「このアラートはなんだったっけ？」ってところから始まる
  - アラートに応じてログを分析する必要がある
    - 起こった事実の確認
- アラートの種類に合わせたログの検索の仕方を思い出すのがつらい
    - 関連情報の収集
  - どこのデータソースにあるのか
  - どういうデータが格納されているのか
  - どういうクエリを書けばいいのか
- 「ドキュメントしておけばいいじゃん」？
  - まず「ドキュメントを読んで理解する」っていうのにハードルがある
  - 時間に余裕があるときならいいけど、隙間時間3分で対応しないといけないときにいちいち見てられる？ 俺は無理だ
